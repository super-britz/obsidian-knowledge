# 知识库维护入口

## 全局偏好

- 使用中文回复。
- 新建和维护的 wiki 正文、索引、日志、README 说明默认使用中文。
- 原文标题、专有技术术语、目录名、URL、Obsidian frontmatter 字段名可以保留英文；必要时用中文解释。
- git commit 时不要添加 Co-Authored-By 或任何暴露 Codex/AI 信息的内容。

## 核心分层

- `raw/`：原始资料层，是事实来源，默认不可变。
- `wiki/`：长期知识层，保存可持续整理、链接和查询的知识页。
- `_meta/`：维护层，保存规则、流程、模板、状态表和校正表，不作为正式知识页。

详细规则见 [[_meta/schema/知识库维护规则|知识库维护规则]]。

## 任务路由

处理任务前，按任务类型读取对应维护文件：

- 普通资料摄取：先读 [[_meta/schema/知识库维护规则|知识库维护规则]] 和 [[wiki/00-索引|00-索引]]。
- 知识库查询：先读 [[wiki/00-索引|00-索引]]，再读相关 wiki 页面和必要 raw 来源。
- 直播文字稿整理：先读 [[_meta/playbooks/直播文字稿整理|直播文字稿整理]]、[[_meta/live/直播课-场次索引|直播课-场次索引]] 和 [[_meta/live/直播课术语校正表|直播课术语校正表]]。
- 新建 wiki 页面：使用 [[_meta/templates/wiki页面模板|wiki 页面模板]]。
- 新建直播场次卡：使用 [[_meta/templates/直播场次整理模板|直播场次整理模板]]。
- 健康检查：按 [[_meta/schema/知识库维护规则|知识库维护规则]] 中的健康检查工作流执行。

## 不可破坏规则

- 除非用户明确要求，不要改写、删减或隐藏 `raw/` 中的原始资料。
- `wiki/` 中的事实、判断和结论必须保留来源；不凭空补全原文没有的信息。
- 维护流程、模板、状态表、术语表放在 `_meta/`，不要混入正式 wiki 知识区。
- `wiki/00-索引.md` 只记录稳定知识入口；临时整理卡、维护模板、状态表不逐条加入总索引。
- `_meta/00-维护日志.md` 按“日期 + 任务主题”合并记录，不写成操作流水账。

## 默认收尾

完成整理或重构后：

1. 检查是否需要更新专题索引或 `wiki/00-索引.md`。
2. 按日志粒度更新 `_meta/00-维护日志.md`。
3. 简短汇报改动、验证结果和仍需用户判断的问题。
